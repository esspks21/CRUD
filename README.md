# JSON CRUD 연락처 관리 시스템

C++17로 작성된 콘솔 기반 CRUD 애플리케이션. 데이터를 `contacts.json` 파일로 영구 저장한다.

---

## 요구사항

| 항목 | 버전 |
|------|------|
| g++ (MinGW) | 6.3 이상 |
| C++ 표준 | C++17 |
| OS | Windows 10/11 |

---

## 빌드 및 실행

```powershell
# 1. 빌드
.\build.ps1

# 2. 실행
.\crud_app.exe
```

> `json.hpp`가 없으면 아래 명령으로 직접 다운로드한다.
> ```powershell
> Invoke-WebRequest -Uri "https://github.com/nlohmann/json/releases/download/v3.11.3/json.hpp" -OutFile json.hpp
> ```

## 메뉴 구조

```
==================================================
        JSON CRUD 연락처 관리 시스템
==================================================
  1. Create  - 연락처 추가
  2. Read    - 전체 목록 보기
  3. Search  - 검색 (ID / 이름)
  4. Update  - 연락처 수정
  5. Delete  - 연락처 삭제
  0. 종료
```

---

## 기능별 사용법

### 1. Create — 연락처 추가

메인 메뉴에서 `1` 입력 후 항목을 순서대로 입력한다.

```
선택: 1

[CREATE] 새 연락처 추가
----------------------------------------
이름     : 홍길동          ← 필수. 비워두면 오류 처리됨
이메일   : hong@test.com
전화번호 : 010-1234-5678
메모     : 개발팀          ← 선택. 그냥 Enter로 건너뛸 수 있음

[완료] ID 1 연락처가 추가되었습니다.
```

- ID는 자동 채번된다 (수동 입력 불필요).
- 이름만 필수이며 나머지 항목은 빈 값도 허용한다.

---

### 2. Read — 전체 목록 보기

메인 메뉴에서 `2` 입력.

```
선택: 2

[READ] 전체 연락처 목록 (3건)
------------------------------------------------------------------------
ID    이름          이메일                   전화번호  메모
------------------------------------------------------------------------
1     홍길동        hong@test.com            010-1234-5678 개발팀
2     김철수        kim@test.com             010-9876-5432 디자인팀
3     이영희        lee@test.com             010-5555-6666
------------------------------------------------------------------------
```

- 등록된 연락처가 없으면 `"등록된 연락처가 없습니다."` 메시지가 출력된다.

---

### 3. Search — 검색

메인 메뉴에서 `3` 입력 후 검색 방식을 선택한다.

#### 3-1. ID로 검색

```
선택: 3

[SEARCH] 검색
1. ID로 검색
2. 이름으로 검색
선택: 1
검색할 ID: 2

검색 결과 1건:
------------------------------------------------------------------------
ID    이름          이메일                   전화번호  메모
------------------------------------------------------------------------
2     김철수        kim@test.com             010-9876-5432 디자인팀
------------------------------------------------------------------------
```

#### 3-2. 이름으로 검색 (부분 일치)

```
선택: 3

[SEARCH] 검색
1. ID로 검색
2. 이름으로 검색
선택: 2
검색할 이름: 김          ← "김"이 포함된 이름 전부 반환

검색 결과 1건:
------------------------------------------------------------------------
ID    이름          이메일                   전화번호  메모
------------------------------------------------------------------------
2     김철수        kim@test.com             010-9876-5432 디자인팀
------------------------------------------------------------------------
```

---

### 4. Update — 연락처 수정

메인 메뉴에서 `4` 입력 → 수정할 ID 입력 → 필드 선택 → 새 값 입력.

```
선택: 4

[UPDATE] 연락처 수정
수정할 ID: 1

현재 정보:
------------------------------------------------------------------------
ID    이름          이메일                   전화번호  메모
------------------------------------------------------------------------
1     홍길동        hong@test.com            010-1234-5678 개발팀
------------------------------------------------------------------------

수정할 필드:
1. 이름     (홍길동)
2. 이메일   (hong@test.com)
3. 전화번호 (010-1234-5678)
4. 메모     (개발팀)
선택: 3
새 전화번호 : 010-0000-9999

[완료] ID 1 연락처가 수정되었습니다.
```

- 한 번에 필드 하나만 수정된다. 여러 필드를 바꾸려면 Update를 반복 실행한다.
- 존재하지 않는 ID 입력 시 `"ID X를 찾을 수 없습니다."` 메시지가 출력된다.

---

### 5. Delete — 연락처 삭제

메인 메뉴에서 `5` 입력 → 삭제할 ID 입력 → `y` 확인.

```
선택: 5

[DELETE] 연락처 삭제
삭제할 ID: 2

삭제할 연락처:
------------------------------------------------------------------------
ID    이름          이메일                   전화번호  메모
------------------------------------------------------------------------
2     김철수        kim@test.com             010-9876-5432 디자인팀
------------------------------------------------------------------------
정말 삭제하시겠습니까? (y/N): y

[완료] ID 2 연락처가 삭제되었습니다.
```

- `y` 또는 `Y` 이외의 입력은 모두 취소로 처리된다.
- 삭제된 ID는 재사용되지 않는다.

## 데이터 파일 형식

실행 파일과 같은 디렉터리에 `contacts.json`이 자동 생성된다.

```json
[
    {
        "email": "hong@test.com",
        "id": 1,
        "memo": "개발팀",
        "name": "홍길동",
        "phone": "010-1234-5678"
    },
    {
        "email": "lee@test.com",
        "id": 3,
        "memo": "",
        "name": "이영희",
        "phone": "010-5555-6666"
    }
]
```

> `contacts.json`을 직접 편집해도 되지만, `id` 값이 중복되지 않도록 주의한다.

## 테스트

ID 중복 발생 여부를 자동으로 검증하는 스크립트가 포함되어 있다.

### Bash (Linux / macOS / Git Bash)

```bash
bash test_id.sh
```

### PowerShell (Windows)

```powershell
powershell -ExecutionPolicy Bypass -File .\test_id.ps1
```

### 테스트 항목

| # | 시나리오 | 확인 내용 |
|---|----------|-----------|
| 1 | 정상 순차 추가 (A → B → C) | ID가 1, 2, 3으로 순서대로 발급되는지 |
| 2 | ID 삭제 후 새 항목 추가 | 삭제된 ID가 재사용되지 않는지 (max+1 전략) |
| 3 | 빈 상태에서 첫 항목 추가 | 첫 ID가 1로 시작하는지 |
| 4 | 외부 편집으로 비연속 ID 삽입 후 추가 | 새 ID가 현재 최대값+1로 발급되는지 |
| 5 | 외부 편집으로 중복 ID 삽입 후 추가 | 앱이 기존 중복을 감지하지 못하는 알려진 한계 확인 |
